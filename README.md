# 🚲 校园共享单车智能调度模型 (MILP) - 核心算法优化与重构

原始优秀论文中提供了基于混合整数线性规划（MILP）的调度模型基础思路，但在实际复现其附录代码时，发现原学术代码为了简化篇幅，省略了许多内容，并且**原模型直接运行会导致以下问题：**
1、求解器直接报错（非线性函数滥用）
2、判定为 Infeasible（无解死锁）
3、产生违背物理常识的“伪优解”

---

## 🛠️ 七大核心算法优化点 (Core Optimizations)

### 1. 消除非线性报错 (Linearization of Absolute Values)
* **原版缺陷：** 目标函数和容量约束中直接使用了 `abs()`，导致求解器无法处理非线性折线，直接抛出 `TypeError`。
* **重构方案：** 引入非负辅助决策变量 `y_abs`，并通过加入双向不等式约束（`y_abs >= y` 且 `y_abs >= -y`），巧妙将绝对值运算转化为纯线性规划，确保求解器顺利解析。

### 2. 大 M 法解耦路径与装载 (Path-Load Decoupling via Big-M)
* **原版缺陷：** 车辆路径变量 $X$ 和装卸量变量 $Y$ 互相独立。导致求解器为了刷分，能在车辆未到达某站点的情况下，产生“远程意念装卸”的荒谬解。
* **重构方案：** 引入大 M 约束 `y_abs <= 容量上限 * X`。从数学上严格锁定：只有车辆到达该节点（X=1），才被允许进行单车装卸（Y>0），否则装卸量必须为 0。

### 3. MTZ 消除“幽灵车辆”子回路 (Subtour Elimination)
* **原版缺陷：** 仅规定了节点的“流入=流出”。求解器为了规避从运维处出发的距离惩罚，会让车辆在校园内部站点之间“凭空出现并无限画圈”，虽然完成了装卸指标，但车辆并未真实出勤。
* **重构方案：** 引入运筹学经典的 **Miller-Tucker-Zemlin (MTZ) 约束**，增加步数辅助变量 `u`。强制要求车辆经过的节点必须呈现严格的先后顺序，彻底掐断了独立子回路的产生。联合“强制休眠约束”，确保所有调度任务必须从运维处始发。

### 4. 严密的供需边界控制 (Strict Supply-Demand Boundaries)
* **原版缺陷：** 缺乏方向与容量限制，导致“贪婪超载”。例如某站点只需拉走 5 辆车，求解器为了自身目标函数最大化，会强行拉走 20 辆车。
* **重构方案：** 根据各站点真实需求差（$Demand$）的正负号，动态锁定变量可行域。车多站点只能拉走（Y≥0），缺车站点只能投放（Y≤0），且总操作量严格 $\le |Demand|$。

### 5. 目标函数引入路径成本惩罚 (Routing Cost Penalty)
* **原版缺陷：** 原目标函数仅追求“装卸量最大化”，导致多条路线装卸量相同时，车辆会随意绕远路。
* **重构方案：** 在目标函数中追加 `- 0.001 * 总行驶距离`。在不影响核心装载指标的前提下，迫使算法自动寻找**最短行驶轨迹**。

### 6. ID 映射修复求解器底层截断 (CBC Solver Encoding Fix)
* **原版缺陷：** PuLP 底层调用 CBC 求解器时，过长的中文变量名（如 `Route_运维处_计算机学院`）会被截断或乱码，导致 Python 无法正确读取求解结果，误报车辆“闲置”。
* **重构方案：** 构建 `N2I` (Name-to-ID) 和 `I2N` 映射字典，全流程使用整数 ID 进行数学推导，输出时再还原为中文，彻底解决解析丢失 Bug。

### 7. 启发式求解参数调优 (Solver Parameter Tuning)
* **原版缺陷：** 引入 MTZ 约束后，由于模型对称性，求解器容易陷入试图证明绝对最优解的“穷举地狱”，求解时间呈指数级爆炸。
* **重构方案：** 引入 `gapRel=0.001` (容差界限) 与 `timeLimit=60`。当算法寻找到的解与理论完美解的差距小于 0.1% 时立即停止搜索。

## 🏆 结果的优越性展示 (Superiority of Results)

经过上述重构，模型在面对 1 个车场、15 个站点、3 辆调度车的复杂 VRP 场景时，展现出了极高的性能与智能逻辑：

1. **极速求解：** 在启用极严密约束的情况下，求解时间从最初的“无法求出”压缩至 **不到 2 秒 (1.89s)** 内即可输出最优解。
2. **完美榨干运力：** 3 辆车准确识别出各自的最优路线，每辆车总负荷精准达到上限 20 辆，整体目标函数得分达到满分状态。
3. **涌现出高级的“人类调度直觉”：**
   * **动态拼车：** 算法能精准将周边零散的小需求（如拉走 1 辆、2 辆、17 辆）无缝拼满一车。
   * **边收边发：** 车辆懂得在同一趟行程中，从 A 站点收车，直接顺路填补给 B 站点。
   * **“路过不摸”逻辑：** 当车辆 3 路过南门时，算法敏锐识别到南门的需求已被车辆 1 满足，因此车辆 3 仅作借道处理，装卸量严格为 `(+0)`，表现出了极高的多兵种协同运筹能力。
