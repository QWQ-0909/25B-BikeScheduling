## 🛠️ 模型工程化优化与 Bug 修复 (Model Refinements)

本项目参考了原优秀论文的 MILP（混合整数线性规划）模型思路，但在复现附录代码时，发现原学术代码在工程落地时存在严重的逻辑漏洞（会导致求解器报错或陷入伪优解）。为此，我们在本项目中引入了以下 **5 项核心改进**，以确保模型具备现实可行性。

### 1. 绝对值非线性报错修复 (Linearization of Absolute Values)
* **原代码缺陷**：原代码在容量约束中直接使用了 Python 的内置函数 `abs(y[i,k,t])`，这在数学规划求解器（如 PuLP/CBC）中属于非法非线性操作，会直接抛出 `TypeError`。
* **修正方案**：引入非负辅助决策变量 `y_abs`，并通过添加双向不等式约束（`y_abs >= y` 且 `y_abs >= -y`），将绝对值成功线性化，确保求解器能够顺利解析约束矩阵。

### 2. 引入大M法绑定路径与装载量 (Path-Load Decoupling Fix via Big-M)
* **原代码缺陷**：原模型中表示车辆路径的变量 $X$ 和表示装卸量的变量 $Y$ 处于完全脱节状态。求解器可以在车辆根本未驶入某节点（$X=0$）的情况下，产生“远程意念装卸”（$Y=20$）的荒谬解。
* **修正方案**：加入大M约束 `y_abs <= 20 * sum(X)`。严格限制：只有当车辆 $K$ 到达节点 $I$ 时，该车辆在该节点的装卸量才被允许大于 0。

### 3. 修复贪婪超额装卸漏洞 (Fixing Greedy Over-loading & Demand Bounds)
* **原代码缺陷**：原模型为了防止 `Infeasible` 报错，删除了微观的供需匹配边界。导致车辆为追求目标函数最大化，会在只需要拉走 5 辆单车的站点，强行拉走 20 辆单车，严重违背物理现实。
* **修正方案**：增加严格的**方向与容量双重约束**。根据 $Demand$ 的正负号锁定变量 $Y$ 的可行域范围（禁止在缺车点拉车，禁止在车多点卸车），并规定 $\sum Y_{ik} \le Demand_i$，确保装卸量绝不超过站点的真实需求。

### 4. 放宽发车强制性避免无解死锁 (Dynamic Idle Vehicle Allowance)
* **原代码缺陷**：通过约束 `sum(x["运维处", j]) == 1`，强行规定所有车辆在每个时间窗都必须离场跑一圈。当部分时段需求量低、用不到所有车辆时，多余车辆无处可去，引发模型无解（Infeasible）。
* **修正方案**：将强制等式 `== 1` 改为不等式 `<= 1`。允许在低谷期部分车辆处于 `X=0` 的闲置待命状态，极大提升了模型应对全天候波峰波谷数据的鲁棒性。

### 5. 目标函数引入路径成本惩罚 (Routing Cost Penalty Integration)
* **原代码缺陷**：原目标函数仅追求“装卸量最大化”。由于缺失对路径成本的度量，一旦装卸量达到上限，求解器会让车辆在校园内随意绕远路甚至走重复子回路（只要满足 30 分钟时长即可）。
* **修正方案**：在目标函数中额外减去一项微小的惩罚项（`0.001 * 路径距离`）。这使得模型在装载收益相同的情况下，会自动寻找**距离最短**的行驶轨迹，产出符合人类直觉的高效调度方案。
